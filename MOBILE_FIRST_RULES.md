# SoulConnect Mobile First Rules

## Product Philosophy

SoulConnect is designed as a mobile application first.

Desktop is only an expanded version of the mobile interface.

Every feature must work perfectly on mobile before desktop.

---

# Devices

Primary

iPhone 15 Pro

393 × 852

Secondary

Pixel 9

Galaxy S25

Desktop adapts.

---

# Layout

Mobile

Single column.

Tablet

Two columns.

Desktop

Three columns.

Cards must stack naturally.

---

# Navigation

Mobile

Sticky Top App Bar

Sticky Bottom Navigation

Desktop

Top Navigation

Collapsible Sidebar

Navigation labels remain consistent.

---

# Bottom Navigation

Home

Community

Check In

Messages

Profile

Never exceed five primary tabs.

---

# Touch Targets

Minimum

48×48px

Large spacing.

Easy one-thumb navigation.

---

# Scrolling

Single vertical scroll.

Avoid nested scrolling.

Use horizontal carousels only where necessary.

---

# Safe Areas

Support

Dynamic Island

Notch

Home Indicator

Android gesture navigation

Respect CSS safe-area insets.

---

# Cards

Full width on mobile.

Comfortable spacing.

Rounded corners.

Large typography.

Readable without zooming.

---

# Forms

Large inputs.

Large buttons.

One field per row.

Keyboard friendly.

---

# Buttons

Primary CTA should always be reachable with the thumb.

Never place important actions only at the top.

---

# Gestures

Support

Swipe

Tap

Long Press

Pull to Refresh

Avoid hover interactions.

---

# Performance

Fast loading.

No layout shift.

Optimize animations.

Avoid heavy GPU usage.

---

# PWA

Installable.

Offline support.

Splash screen.

Standalone mode.

Native app feeling.

---

# Final Quality Check

Before shipping any screen ask:

Can someone comfortably use this with one hand?

If not,

redesign it.
